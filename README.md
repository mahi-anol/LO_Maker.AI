# 📚 পাঠ পরিকল্পনা জেনারেটর — Lesson Plan Generator

An AI-powered lesson plan generator for Bengali-medium classrooms.
Built with **LangGraph + LangChain + OpenAI GPT** and deployed as a **Gradio app on HuggingFace Spaces**.

---

## ✨ Features

- 📖 **RAG from Textbook** — Upload any textbook PDF; the app uses vector search to find relevant content for the Learning Outcome
- 🤖 **LangGraph Pipeline** — Each lesson section (Launch, Explore, Conceptualize, Guided Practice, Independent Practice, Closing) is generated by a dedicated graph node
- 🇧🇩 **Bengali Output** — All Bangla content generated matches the provided template structure
- 📄 **PDF Output** — Formatted with blue headers, tables, and Bengali font (Noto Sans Bengali)
- 📝 **DOCX Output** — Word document with matching table styling
- 🎛️ **Model Choice** — Users can select GPT-4o, GPT-4o-mini, GPT-4-Turbo, or GPT-3.5-Turbo

## 🚀 Deployment on HuggingFace Spaces

### Step 1 — Create a new Space
1. Go to [huggingface.co/spaces](https://huggingface.co/spaces)
2. Click **New Space**
3. Select **Gradio** as the SDK
4. Set Python version to **3.10** or higher

### Step 2 — Upload files
Upload all files from this project:
```
app.py
pipeline.py
pdf_generator.py
docx_generator.py
font_manager.py
requirements.txt
README.md
```

### Step 3 — Set the API key secret
1. In your Space, go to **Settings → Repository secrets**
2. Add a new secret:
   - **Name:** `OPENAI_API_KEY`
   - **Value:** your OpenAI API key (starts with `sk-...`)

### Step 4 — Deploy
The Space will auto-build and deploy. The first run downloads the Bengali font (~2MB).

## 🔄 How It Works

```
User Input (LO + Textbook PDF)
        │
        ▼
[Node 1] Build FAISS vector store from textbook PDF
        │
        ▼
[Node 2] Retrieve relevant textbook content for the LO
        │
        ▼
[Nodes 3–10] Generate each lesson section in parallel conceptually
   ├── Lesson Vision (Why academic, Why non-academic, What)
   ├── Key Points (How steps)
   ├── Assessment (MCQ + calculation question + exemplar)
   ├── Launch (SEL + curiosity + prior knowledge)
   ├── Explore (activity + probing questions)
   ├── Conceptualize (Do + Say + Example + CFU)
   ├── Guided Practice (problem + WTD + solution)
   ├── Independent Practice (problem + WTD)
   └── Lesson Closing (exit ticket + closing)
        │
        ▼
[Node 11] Assemble all sections
        │
        ▼
PDF Generator (ReportLab) → lesson_plan.pdf
DOCX Generator (python-docx) → lesson_plan.docx
```

## 📝 Template Structure

The generated lesson plan follows this template (matching the provided sample):

| Section | Content |
|---------|---------|
| **Header** | Teacher name, Subject, Grade, Time |
| **Lesson Vision** | What / Why (Academic + Non-Academic) / How |
| **Assessment** | MCQ + Calculation question + Exemplar answer |
| **Launch** (5 min) | SEL mood check + Sparking curiosity + Prior knowledge |
| **Explore** (7 min) | Exploration activity + Probing questions |
| **Conceptualize** (8 min) | Do + Say + Worked example + CFU |
| **Guided Practice** (10 min) | Problem + WTD + Teacher solution |
| **Independent Practice** (8 min) | Solo problem + WTD |
| **Lesson Closing** (7 min) | Exit ticket + Summary |

## 🐛 Troubleshooting

**Bangla text not rendering in PDF?**
→ The app downloads Noto Sans Bengali at startup. Check the Spaces logs.
→ If font download fails, text will appear as boxes — place `NotoSansBengali-Regular.ttf` in the `fonts/` folder manually.

**OpenAI API error?**
→ Verify `OPENAI_API_KEY` is set in HuggingFace Secrets.
→ Check your OpenAI account has sufficient credits.

**PDF generation takes too long?**
→ The pipeline makes ~10 separate GPT API calls. GPT-4o-mini is significantly faster.
